# ADR-0004: 终端环境变量仅对新会话生效

- **状态**: 已接受
- **日期**: 2026-06-14
- **决策者**: 产品方 + AI 架构师
- **关联**: Q30

## 上下文

Natives 支持多组环境配置（如"工作"、"个人"），终端启动时自动注入当前激活配置的环境变量。用户可能在运行中的终端会话期间切换环境配置。

## 决策

环境变量注入仅对新创建的终端会话生效，已运行的会话保持原有环境。

## 理由

1. **避免运行中断** — 正在执行的命令可能依赖当前环境变量，突然变更会导致命令失败或行为异常
2. **与主流终端一致** — VS Code、iTerm2、macOS Terminal 都不会在运行中修改已打开终端的环境
3. **用户可预期** — 用户理解"新开终端用新配置"的模型

## 用户体验

- 切换环境配置后，终端区域显示提示："新配置已生效，新开终端标签将使用新配置"
- 用户可通过快捷键新开终端标签（Cmd+T）
- 已运行的终端标签不受影响

## 影响

- **Environment Injector（M8）**：`injectEnv()` 仅在 `createSession()` 时调用
- **Terminal UI**：切换配置时显示提示信息
